# UI Spec - Student Exchange

Updated: 2026-02-16

## Design Direction
- Clean admin/customer separation.
- Tailwind-based reusable UI (`Button`, `Input`, `Card`, `Badge`).
- Mobile-first layouts.

## Public Routes
- `/login`
- `/products`
- `/products/:id`
- `/cart`
- `/checkout`
- `/order-success/:orderCode`
- `/events`
- `/events/:id`
- `/listings` (authenticated user)
- `/support`
- `/support/faq`
- `/support/contact`
- `/support/track-order`
- `/support/policies`
- `/support/policies/purchase`
- `/support/policies/refund`

## Admin Routes
- `/admin/login`
- `/admin`
- `/admin/listings`
- `/admin/listings/new`
- `/admin/listings/:id/edit`
- `/admin/orders`
- `/admin/events`
- `/admin/tickets`

## Screen Requirements

### Listings / Cart / Checkout
- Listing grid with search + category + pagination.
- Listing detail with quantity and add-to-cart.
- Cart allows quantity update/remove.
- Checkout submits customer info and redirects to success page.

### Events
- Event list/detail from backend.
- Event registration form.
- Admin event page supports CRUD + registration list.

### Support
- FAQ page with search + category.
- Contact page creates ticket and shows ticket code.
- Track page supports both order and ticket tracking.

### Admin
- Dashboard with summary metrics.
- Listings table + form.
- Orders table with status transition actions.
- Tickets table with status update + reply.

## Access Rules
- Unauthenticated users redirected to `/login` for protected user routes.
- Admin routes require `ADMIN` role; otherwise redirect.
