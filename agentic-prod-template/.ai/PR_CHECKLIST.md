# PR Checklist

1. Task ID and title (for example `T021 - ...`)
2. What changed (1-3 sentences)
3. Why it changed (business/bug reason)
4. How to test (exact commands and expected result)
5. Risk areas (data loss, security, backwards compatibility)
6. Rollback plan
7. Docs/tasks updated (`docs/00-context.md`, `tasks/task-queue.md`, `tasks/done.md` when applicable)
